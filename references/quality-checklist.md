# Quality Checklist & Anti-Hallucination Rules

## 分析前检查

### 输入评估
- [ ] 文件可正常读取（非损坏的 PDF）
- [ ] 记录了原文行号（用于后续引用）
- [ ] 识别了 AI4S 领域（医疗影像、药物发现、蛋白质科学等）

---

## 引用质量检查

### 原文引用三要素
- [ ] 所有证据性论点都有原文引用
- [ ] 引用使用原文英文（未改写或转述）
- [ ] 标注了准确的行号或行号范围
- [ ] 标注了原文位置（section/subsection）

### 引用格式
- [ ] 长段落使用引用块格式：`> 原文（行号）【位置】`
- [ ] 短句使用行内格式：`作者认为「原文」（行号，位置）`
- [ ] 原文片段用 `「...」` 高亮包裹

---

## 内容质量检查

### 事实准确性
- [ ] 元信息（标题、作者、年份）直接来自原文
- [ ] 数值结果原文照抄，不四舍五入或改写
- [ ] 作者单位来自论文的作者部分，不猜测

### 不确定性标记
- [ ] 缺失字段标注 `[未明确给出]`，不空白或猜测
- [ ] 有歧义信息标注 `[不确定]` 并说明歧义原因
- [ ] 跳过的章节明确说明原因

### 领域假设检查
- [ ] AI4S 领域分类基于论文内容，不假设
- [ ] 方法分析符合 AI4S 领域特征
- [ ] 无通用 AI/ML 术语注入非 AI4S 论文

---

## 常见幻觉模式

| 模式 | 错误做法 | 正确做法 |
|---------|-----------|----------|
| 场所猜测 | "发表于 Nature"（原文未提）| `[未明确给出]` |
| 贡献膨胀 | 增加论文未声明的贡献 | 仅列出论文声明的贡献 |
| 结果外推 | "达到所有基准的 SOTA" | 引用确切基准和指标 |
| 作者专长 | "计算机视觉专家" | 仅陈述论文声明的内容 |
| 未来工作编造 | 发明未来方向 | 仅引用论文自己的未来工作部分 |
| 方法漏洞填充 | 猜测架构细节 | 标注为 `[未明确给出]` |

## 双文输出检查

### 中文版
- [ ] 分析内容使用中文表达
- [ ] 技术术语保留英文（Transformer、GNN、NORS 等）
- [ ] 原文引用使用混合格式，保持英文原文
- [ ] 符合中文学术表达规范

### 英文版
- [ ] 内容与中文版完全对应
- [ ] 使用英文学术规范
- [ ] 原文引用保持英文原文（不翻译）
- [ ] 行号和位置标注与中文版一致

### 文件命名
- [ ] 中文版：`[论文标题]_review_ch.md`
- [ ] 英文版：`[论文标题]_review_en.md`
- [ ] 保存到指定路径（或当前目录）

## 10 维度分析检查

### 维度 1：核心问题提取
- [ ] 概括了核心研究问题
- [ ] 说明了瞄准的学术/技术缺口
- [ ] 解释了问题重要性（非背景介绍）

### 维度 2：理论基准锚定
- [ ] 识别了核心技术范式
- [ ] 列出了奠基性工作
- [ ] 说明了创新点与基础工作的区别

### 维度 3：方法论拆解
- [ ] 描述了模型架构类型
- [ ] 说明了数据来源和规模
- [ ] 列出了训练配置（损失函数、优化器等）
- [ ] 说明了评估方法和基线对比

### 维度 4：实证证据审查
- [ ] 说明了假设验证结果
- [ ] 列出了 3 个最关键证据
- [ ] 提供了定量的基线对比

### 维度 5：方法论漏洞查找
- [ ] 识别了 2-3 个致命漏洞
- [ ] 说明了漏洞类型和潜在影响
- [ ] 提供了改进建议

### 维度 6：边界条件与局限
- [ ] 界定了数据边界
- [ ] 说明了技术边界
- [ ] 列出了已知局限
- [ ] 补充了额外观察局限

### 维度 7：文献交叉验证
- [ ] 说明了反驳或拓展的研究
- [ ] 评估了战略地位（突破性/渐进性/验证性）

### 维度 8：核心贡献提炼
- [ ] 列出了 2 点核心贡献
- [ ] 说明了贡献的影响
- [ ] 评估了是否带来范式转变

### 维度 9：未来研究裂变
- [ ] 提出了 3 个新研究方向
- [ ] 每个方向包含研究问题和方法升级

### 维度 10：可复现性评估
- [ ] 评估了代码可得性
- [ ] 评估了数据可得性
- [ ] 评估了文档完整性
- [ ] 给出了复现难度评级

---

## 输出前自查

在生成最终报告前，验证：

- [ ] 每个不确定字段标注了 `[不确定]` 或 `[未明确给出]`
- [ ] 每个贡献标注了 `[原文声明]` 或 `[模型归纳]`
- [ ] 没有静默跳过章节 — 跳过的章节说明了原因
- [ ] AI4S 领域分类与论文内容匹配
- [ ] 所有引用都有行号和位置标注
- [ ] 中英双文结构一致
- [ ] 文件命名符合规范
